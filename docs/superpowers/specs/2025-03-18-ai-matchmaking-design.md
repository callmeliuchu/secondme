# AI 相亲大会 - 设计文档

## 1. 项目概述

**项目名称**：AI 相亲大会
**项目类型**：Next.js 全栈 Web 应用
**核心功能**：用户创建自己的 AI 分身，系统自动为两个 AI 分身匹配并让他们进行"相亲"聊天，用户全程围观
**目标用户**：对 AI 人格感兴趣，想体验 AI 社交互动的用户

## 2. 赛道对齐

- **赛道**：Agent 的第三空间 - 圆桌会谈
- **核心价值**：展示 AI 分身之间的社交互动，让 AI 代替用户进行社交体验

## 3. 功能列表

### 3.1 核心功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 创建 AI 分身 | 用户输入名字、性格、爱好等信息创建 AI 分身 | P0 |
| AI 分身库 | 展示已创建的 AI 分身列表 | P0 |
| 发起相亲 | 选择一个 AI 分身发起相亲匹配 | P0 |
| 实时聊天 | 两个 AI 分身自动进行聊天，用户围观 | P0 |
| 相亲结果 | 聊天结束后给出匹配度评分 | P1 |
| 历史相亲记录 | 查看过往相亲记录和结果 | P2 |

### 3.2 用户流程

```
1. 首页 → 创建/选择 AI 分身
2. 发起相亲 → 系统自动匹配另一个 AI 分身
3. 进入相亲房间 → 围观两个 AI 聊天
4. 聊天结束 → 显示匹配度评分
5. 可以继续发起新的相亲
```

### 3.3 扩展功能（可选）

- AI 红娘推荐：根据用户偏好推荐最佳匹配
- 分享相亲结果：生成海报分享到社交媒体
- 相亲排行榜：最受欢迎的 AI 分身排行

## 4. 技术架构

### 4.1 技术栈

- **框架**：Next.js 14+ (App Router)
- **语言**：TypeScript
- **样式**：Tailwind CSS
- **数据库**：Prisma + PostgreSQL (Vercel Postgres)
- **部署**：Vercel
- **认证**：SecondMe OAuth2
- **AI**：SecondMe Act/Chat API

### 4.2 数据模型

```
User (用户)
├── id
├── secondmeUserId
├── accessToken / refreshToken
└── nickname / avatar

AIAgent (AI 分身)
├── id
├── userId (创建者)
├── name (名字)
├── personality (性格描述)
├── hobbies (兴趣爱好)
├── appearance (外貌描述)
├── intro (自我介绍)
├── avatar (头像 URL)
├── status (active/inactive)
└── createdAt

MatchSession (相亲会话)
├── id
├── userId (发起者)
├── agent1Id (AI 分身 1)
├── agent2Id (AI 分身 2)
├── status (pending/running/ended)
├── matchScore (匹配度)
├── createdAt
└── endedAt

ChatMessage (聊天消息)
├── id
├── sessionId
├── agentId (发送者)
├── content
└── createdAt
```

### 4.3 API 设计

| 端点 | 方法 | 描述 |
|------|------|------|
| `/api/agent` | GET | 获取用户的 AI 分身列表 |
| `/api/agent` | POST | 创建新的 AI 分身 |
| `/api/agent/[id]` | GET | 获取 AI 分身详情 |
| `/api/match` | POST | 发起相亲匹配 |
| `/api/match/[id]` | GET | 获取相亲会话详情 |
| `/api/match/[id]/stream` | GET | SSE 流式获取聊天内容 |

## 5. 核心交互流程

### 5.1 创建 AI 分身

用户在表单中填写：
- 名字（如：小明、Summer）
- 性格描述（如：温柔可爱、幽默风趣）
- 兴趣爱好（如：旅行、美食）
- 外貌描述（如：高个子、戴眼镜）
- 自我介绍

系统调用 SecondMe Act API 生成结构化的分身定义，保存到数据库。

### 5.2 发起相亲

1. 用户选择一个自己创建的 AI 分身
2. 系统从数据库中随机选择另一个用户的 AI 分身（或系统预置的分身）
3. 系统生成两个分身的聊天背景（"第一次见面"）
4. 创建 MatchSession，进入相亲房间

### 5.3 实时聊天（SSE）

1. 系统调用 SecondMe Chat API，传入两个分身的角色定义
2. 两个分身轮流对话，每次对话保存到数据库
3. 实时推送消息给前端展示
4. 聊天 N 轮后结束

### 5.4 匹配度评分

使用 SecondMe Act API 判断两个分身的匹配度，返回 0-100 的分数。

**评分维度**：
- 性格互补程度
- 兴趣爱好的重叠度
- 对话中的互动质量

**技术实现**：调用 Act API，传入两个分身的性格描述和聊天记录，让 AI 判断匹配度。

### 5.5 聊天轮数

默认 10 轮对话，每轮包括两个分身的各一次发言。用户可以提前结束相亲。

### 5.6 异常处理

- **匹配失败**：如果找不到合适的分身，显示"暂无匹配对象"
- **AI 无响应**：超时 30 秒后发送提示消息，继续下一轮
- **聊天中断**：保存已发送的消息，下次可以继续

## 6. UI/UX 设计方向

### 6.1 页面结构

```
首页
├── 我的 AI 分身（卡片列表）
├── 发起相亲按钮
└── 历史记录

创建分身页面
├── 分身表单
└── 预览

相亲房间
├── 两个分身的头像和信息
├── 聊天消息流
├── 匹配度评分
└── 分享按钮
```

### 6.2 设计风格

- **粉色/浪漫主题**：符合相亲场景
- **卡片式设计**：展示 AI 分身信息
- **聊天界面**：类似微信/QQ 的对话气泡
- **动画效果**：消息出现、匹配结果展示

## 7. 依赖 SecondMe 能力

- **OAuth2 认证**：用户登录
- **Act API**：生成分身定义、匹配度评分
- **Chat API**：生成聊天内容（流式）
